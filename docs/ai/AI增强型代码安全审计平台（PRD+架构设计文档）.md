# 第一部分：产品需求文档（PRD）

## 1\. 产品概述

### 1\.1 产品定位

搭建**传统 SAST 静态扫描 \+ 代码图谱分析 \+ AI 大模型语义审计**一体化安全审计平台。
解决传统代码审计三大行业痛点：

1. 传统 SAST 误报极高、逻辑漏洞大面积漏报；

2. 人工审计效率低、新人上手慢、业务逻辑漏洞依赖资深安全；

3. 漏洞缺少自动验证、自动归因、自动修复，无法形成运营闭环。

平台目标：**安全左移自动化、逻辑漏洞智能化、漏洞运营体系化**。

### 1\.2 核心能力范围

1. 多语言代码资产统一接入与管理

2. 传统多引擎 SAST 静态扫描（规则漏洞 \+ 依赖漏洞）

3. 代码 CPG 属性图构建、污点数据流分析

4. AI\-RAG 私有知识库智能审计

5. AI 误报降噪、AI 逻辑漏洞挖掘、AI 自动 POC 验证

6. AI 智能修复与补丁生成

7. 漏洞全生命周期工单管理

8. CI/CD 流水线安全卡点

9. 安全风险大盘、合规报表、漏洞运营分析

### 1\.3 目标用户

- 安全审计工程师：替代人工重复甄别、深度逻辑审计辅助

- 研发工程师：提交代码自动检测、快速修复指引

- 研发负责人：项目安全风险统计、质量管控

- 安全管理者：合规审计、风险运营、漏洞趋势分析

## 2\. 整体业务需求

### 2\.1 业务目标

1. 静态告警**误报率降低 80%\+**

2. 业务逻辑漏洞检出率提升 40%

3. 代码审计人效提升 60%

4. 实现代码提交、合并、上线全流程安全卡点

5. 漏洞从发现、定级、分配、修复、复测、归档**全自动化闭环**

### 2\.2 支持技术栈

后端：Java、Go、PHP、Python
前端：Vue/React/JS/TS
依赖解析：Maven、Gradle、NPM、Pip、Composer
仓库源：GitLab、Gitee、GitHub

## 3\. 功能需求详述

### 3\.1 资产接入模块

1. 支持 HTTP/Token 方式接入私有代码仓库

2. 自动拉取项目、分支、Commit 记录

3. 支持手动触发全量扫描、WebHook 增量触发扫描

4. 项目按业务线、团队、外包 / 自研分类管理

5. 源码**容器临时拉取、不持久化存储**，保障源码安全

### 3\.2 传统 SAST 多引擎扫描模块

1. 集成开源引擎：SonarQube、CodeQL、gosec、Bandit

2. 集成依赖漏洞引擎：Dependency\-Check、Snyk

3. 内置自研 AST 规则引擎，支持自定义安全规则

4. 扫描模式：

    - 增量扫描：仅本次 diff 变更代码

    - 全量扫描：整仓完整审计

5. 输出标准化漏洞字段：文件、行号、漏洞类型、风险等级、规则 ID、触发条件

### 3\.3 代码图谱构建模块（AI 前置依赖）

1. 基于 Tree\-sitter/AST 构建抽象语法树

2. 构建 CPG 代码属性图（控制流 \+ 数据流 \+ 函数调用图）

3. 污点 Source/Sink 链路追溯

4. 自动提取高危代码片段、危险函数、可控入参点位

5. 为 AI 提供结构化代码语义数据，解决大模型幻觉、超长上下文问题

### 3\.4 AI\-RAG 知识库模块

#### 3\.4\.1 知识库数据源

- 公共安全知识库：CWE 标准、OWASP Top10、漏洞评级规范

- 企业私有知识库：历史审计漏洞、修复案例、误报样本

- 框架安全库：Spring、Dubbo、MyBatis、Vue 安全规范

- 业务漏洞库：越权、金额篡改、营销漏洞、权限漏洞样本

#### 3\.4\.2 RAG 能力

1. 向量化存储、语义检索

2. 审计时自动召回相似漏洞、历史修复方案

3. 持续增量学习新漏洞样本

### 3\.5 AI 核心审计能力（核心差异化需求）

#### 3\.5\.1 AI 误报降噪

1. 对 SAST 原始告警批量智能甄别

2. 自动区分：真实漏洞 / 框架防护 / 不可达代码 / 误报

3. 输出降噪结论与判断依据

#### 3\.5\.2 AI 业务逻辑漏洞挖掘

自动识别传统工具无法扫描的漏洞：

1. 水平 / 垂直越权、数据归属未校验

2. 金额、数量、优惠参数前端可控后端未校验

3. 活动次数无幂等、无限领取、重复下单

4. 验证码可复用、无过期、可爆破

5. 单据状态绕过、归档数据篡改

6. 接口前置流程缺失校验

#### 3\.5\.3 AI 漏洞归因与链路分析

1. 自动梳理完整污点传播链路

2. 定位漏洞根因（编码问题 / 设计缺陷 / 配置问题）

3. 判定漏洞可利用门槛、影响范围

#### 3\.5\.4 AI 自动 POC 生成与验证

1. 基于CodeX语法解析能力，根据漏洞场景、代码逻辑生成标准化可复现 POC 脚本

2. 提交隔离沙箱自动验证漏洞真实性、POC可用性

3. 真实漏洞保留、误报自动过滤，同步记录CodeX校验日志

#### 3\.5\.5 AI 自动修复补丁生成

1. 基于CodeX生成标准修复代码、替换片段，保证代码可编译、语法合规

2. 结合LLM输出修复说明、安全编码规范、业务适配注意事项

3. 支持批量同类漏洞批量修复、统一合规整改，适配企业编码规范

### 3\.6 人工审计工作台

采用 \\*\\*「确定性静态扫描 \+ 结构化代码图谱 \+ LLM 语义推理 \+ RAG 知识库增强」混合架构 \\*\\*
核心设计理念：

1. 传统 SAST 保证**规则漏洞稳定、精准、可卡点**

4. AI 大模型解决**逻辑漏洞、误报甄别、智能运营**问题

**禁止纯 AI 审计**，采用「先结构化、后智能推理」的工业级落地架构。

## 3\. 核心业务数据流（完整闭环）

1. 开发提交代码 → WebHook 触发

2. 调度中心创建扫描任务 → 拉起独立 Docker 沙箱

3. 沙箱拉取代码 → 执行 SAST 扫描 \+ 代码图谱解析

4. 提取可疑漏洞片段、污点链路、代码结构

5. RAG 召回历史漏洞与规范样本

6. CodeX Agent 主导代码层级审计、语法校验、漏洞精准定位，RAG提供规范依据，LLM辅助业务语义推演

7. CodeX Agent 完成核心能力：误报过滤、逻辑漏洞识别、风险定级、根因定位

8. CodeX Agent 自动生成可直接运行的标准化POC、可编译无报错修复补丁

9. 生成漏洞工单、推送责任人

10. 人工终审、修复复测、漏洞闭环

11. 新漏洞样本自动入库，迭代更新规则库

## 4\. 核心模块详细架构设计

### 4\.1 沙箱安全隔离架构

1. 每次扫描独立容器，资源配额限制

2. 容器仅临时拉取代码，无外网出网权限

3. 扫描结束**容器销毁、源码清零**

4. 杜绝源码外泄、落地留存

### 4\.2 AI\-RAG 架构细节

1. 离线批量构建企业安全向量库

2. 在线审计实时 Top\-K 召回

3. 召回内容 \+ 代码片段 \+ Prompt 工程送入大模型

4. 输出带依据、带参考、低幻觉的审计结论

5. 新漏洞持续增量入库

### 4\.3 多 Agent 协同架构

各司其职、解耦设计：

- 理解 Agent：负责陌生项目架构梳理、攻击面识别

- 降噪 Agent：负责百万级告警清洗

- 挖掘 Agent：专注业务逻辑漏洞语义分析

- 修复 Agent：专注验证、POC、补丁生成

### 4\.4 核心架构：CodeX AI Agent 主控 \+ LLM 辅助增强架构 \& 降级策略

#### 4\.4\.1 核心设计理念（CodeX 作为 AI Agent 核心驱动）

摒弃传统LLM主导的审计模式，平台采用**CodeX AI Agent 为主控、通用LLM为辅助**的工业级智能审计架构。CodeX 是专为代码理解、代码推理、代码工程落地设计的智能体，具备自主任务拆解、代码分析、漏洞研判、工程校验、结果输出的完整自主能力，解决通用大模型代码推理不准、生成代码不可编译、漏洞研判脱离工程实际的核心痛点。通用LLM仅作为语义辅助，负责业务场景解读、归因总结、报告输出，不参与核心代码审计决策。

**双模型权责边界（严格隔离）**

#### 4\.4\.1 CodeX 模型集成核心设计

平台采用**通用语义大模型 \+ CodeX 代码专用模型**双引擎融合架构，差异化分工、互补赋能，解决通用大模型代码解析不精准、修复代码兼容性差、语法识别误差大的核心问题。CodeX 专注代码语法解析、代码生成、补丁修复、代码合规校验；通用LLM专注业务语义推理、漏洞归因、逻辑分析、自然语言总结，双模型协同完成全流程智能审计。

**模型分工机制：**

- CodeX 擅长域：结构化代码解析、AST语法校验、漏洞代码片段精准定位、可编译修复补丁生成、POC脚本标准化输出、代码规范校验、多语言语法兼容适配

- 通用LLM擅长域：业务逻辑梳理、漏洞根因分析、误报逻辑推演、风险定级解读、审计报告撰写、复杂业务场景推理

#### 4\.4\.2 双模型联动调用流程

1. 前置输入：SAST引擎、CPG图谱输出结构化代码片段、污点链路、漏洞基础信息

2. 第一层处理：CodeX优先接管，精准解析代码语法、校验漏洞真实性、生成标准化修复代码片段

3. 第二层处理：通用LLM结合RAG知识库、业务场景，对CodeX输出结果做业务层校验、归因分析、风险定级

4. 结果融合：平台聚合双模型输出，生成「语法合规\+业务适配」的最终审计结论、修复方案、POC脚本

5. 结果校验：CodeX二次校验修复代码可编译性、无语法BUG，杜绝无效补丁

#### 4\.4\.3 分级降级策略（保障生产稳定）

1. 双模型正常：CodeX\+LLM协同审计，输出最高精度结果
