## Issues Fixed During Verification

1. **`vue-echarts` install failure** — the package ships a broken `vue-demi` symlink under macOS node 20.
   - **Fix:** `npm install --install-strategy=nested` resolves the conflict.
2. **`@vue/server-renderer` missing** — `@vue/test-utils@2.4.6` requires it transitively; not auto-installed.
   - **Fix:** `npm install --save-dev @vue/server-renderer` (added to `package.json`).
3. **`monaco-editor` is ESM-only with no main field** — vitest could not resolve it.
   - **Fix:** added `vite.config.ts` alias `monaco-editor → tests/stubs/monaco.ts` for test mode; production build unaffected.
4. **TypeScript: `Editor` vs `editor` import** — `@guolao/vue-monaco-editor` exports the type as `Editor`, not `editor`.
   - **Fix:** updated both `CodeViewer.vue` and `FixSnippetEditor.vue` imports.
5. **TypeScript: `revealLineInCenterIfOutside` not in monaco IStandaloneCodeEditor** — actual method is `revealLineInCenterIfOutsideViewport`.
   - **Fix:** renamed the call in `CodeViewer.vue`.
6. **TypeScript: `<Table<Vuln>` template syntax** — generic type params are not valid in Vue templates.
   - **Fix:** removed the generic from the template; the typed `tableColumns` array already constrains the data.
7. **TypeScript: generic template `K extends` in `ExploitConditionForm`** — Vue compiler cannot narrow `K` inside the emit signature.
   - **Fix:** explicit `type FieldKey = ...` + switch dispatch.
8. **TypeScript: AntD `ItemType` discriminated union** — `SubMenuType` requires `children`, so the menu-item factory expression failed to narrow.
   - **Fix:** wrap each icon component with `h()` to produce a `VNode`; this also matches AntD's runtime expectations.
9. **TypeScript: `MenuProps['items']` resolves to the same `ItemType[]`** — same issue.
   - **Fix:** same as #8.
10. **TypeScript: AntD `Switch` and `Select` onChange use `(value, option)` not `(v: boolean)`** — our closures' type assertions didn't match the overload.
    - **Fix:** accept `unknown` and cast with `Boolean(...)` / `Array.isArray(...)`.
11. **Vitest: `monaco-editor` cannot be loaded** — file-level `vi.mock` did not help because Vite resolves the package entry first.
    - **Fix:** combined approach — vite alias to stub + module-level `vi.mock` for the wrapper.
